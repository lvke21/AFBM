Core Navigation Prioritization

Die Navigation wurde von einer gleichwertigen Systemliste zu einer aufgabenorientierten Manager-Navigation umgewichtet. Neue Nutzer sehen jetzt die wichtigsten Entscheidungen zuerst.

Status: Gruen

UX-Entscheidung

Die Sidebar gruppiert nicht mehr nach internen Bereichen wie Team, Game, Finance und Development, sondern nach Nutzungsabsicht:

  • Core Actions: Dashboard, Game Flow, Roster, Depth Chart, Contracts/Cap, Development.
  • Team Management: Team Overview, Trade Board, Inbox.
  • Analysis/Secondary: Finance, League, Draft, Savegames.

Core Actions erhalten mehr visuelles Gewicht. Trade Board bleibt sichtbar, ist aber als Planungsflaeche statt als Hauptabschluss-Flow eingeordnet.

Responsive Verhalten

Die Sidebar nutzt die bestehenden Shell-Layouts weiter. Die Team-Section-Navigation ist jetzt in drei Gruppen organisiert und stackt responsiv: auf kleineren Breiten bleiben Core Actions zuerst sichtbar, waehrend Planning und Analysis darunter folgen.
