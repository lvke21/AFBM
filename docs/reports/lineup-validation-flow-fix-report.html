Lineup Validation Flow Fix

Ziel erreicht: Der Testkader kann mehrere Wochen spielen, ohne dass fehlende Secondary-Starter-Slots den Match Start blockieren. Fehlende Positionen werden als schwache Ersatzspieler simuliert, im Preview als Warning angezeigt und im Match Report als Weak Position Group sichtbar.

Core blockiertQB, RB, WR bleiben harte Mindestbasis.
Secondary warntFB, OL, Defense, ST blockieren nicht mehr.
Auto-Fill 62 OVRVirtuelle Dev-Spieler druecken Simulation.
3 Wochen gruenWoche 1-3 starten, laufen live und reporten.

Geaenderte Validierungslogik

Ebene Vorher Nachher
Core Positions Alle offenen Starter-Slots wurden gleich behandelt. Nur fehlende Core-Starter wie QB, RB, WR koennen hart blockieren.
Secondary Positions Fehlende Slots wie FB, LG, C, RG, RT, LE, FS, LS verhinderten den Start. Fehlende Secondary-Slots erzeugen Lineup Warning, bleiben aber spielbar.
Game Preview Match Start gesperrt trotz READY Week State. Match starten mit Lineup-Warnung und Hinweis: Lineup unvollstaendig - Performance kann leiden.
Simulation Fehlende Slots waren primaer UI-Blocker. Fehlende Secondary-Slots erzeugen negative Rating-Modifikatoren wie schwache 62-OVR-Replacements.
Match Report Der Nutzer sah den Blocker, aber keine spielerische Konsequenz. Report zeigt Weak Position Group mit den Auto-Fill-Positionen und Leistungsabzug.

Auto-Fill Beispiel

Fehlender LS-Starter erzeugt intern einen virtuellen Spieler: dev-auto-fill-ls-1, Name Dev Auto-Fill LS, Position LS, 62 OVR, Tag DEV_AUTO_FILL. Der Spieler wird nicht als echter Franchise-Spieler verkauft, sondern dient als Simulationsersatz fuer unterbesetzte Secondary-Gruppen.

Vorher / Nachher Flow

Flow Schritt Vorher Nachher
Woche 2 Game Preview READY, aber Start gesperrt wegen 12 Starter-Slots. READY, Start erlaubt mit Warning und 12 Auto-Fill-Slots.
Schlechtes Lineup spielen Nicht moeglich. Moeglich; Nutzer akzeptiert bewusst Performance-Risiko.
Report Feedback Kein Spiel, also kein Lerneffekt. Report nennt unterbesetzte Gruppen und Auto-Fill-Effekt.

Konkrete Validierung

  • Woche 1: gestartet, Live, Report, Score 15 : 13, Record BOS 1-0.
  • Woche 2: gestartet, Live, Report, Score 3 : 16, Record BOS 2-0.
  • Woche 3: gestartet, Live, Report, Score 7 : 0, Record BOS 3-0.
  • Alle drei Previews zeigten Match starten mit Lineup-Warnung statt hartem Blocker.
  • Alle drei Reports enthielten Weak Position Group bzw. unterbesetzte Gruppen.

Report-Beispiel

Im Spielbericht wird sichtbar: FB, LG, C, RG, RT, LE, RE, LOLB, ROLB, FS, SS, LS wurden mit schwachen Ersatzspielern um 62 OVR simuliert und drueckten die Teamleistung. Damit wird aus einem vormals technischen Blocker ein spielerisches Risiko.

Entscheidung

Flow-Blocker behoben. Der Testkader bleibt minimal und unvollstaendig, aber der Nutzer kann jetzt bewusst mit schlechtem Lineup spielen. Die Konsequenz erscheint in Simulation und Report statt als unloesbare UI-Sperre.
