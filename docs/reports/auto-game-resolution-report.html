Verbunden

  • startGameForUser erzeugt Drives, Score und Team-Match-Stats.

  • TeamSeasonStat wird aus finalem Score und Match Stats aktualisiert.

Status Flow

  • SCHEDULED: Match wartet auf Start.
  • IN_PROGRESS: Simulation ist erzeugt und Score gespeichert.
  • COMPLETED: Ergebnis ist fixiert, Report und Record sind bereit.

Integritaetsbewertung

Keine UI Override-Moeglichkeit

Der Live Screen enthaelt keine Score-Felder. Der Server Action Contract liest nur saveGameId und matchId.

Systemgesteuertes Ergebnis

Prisma nutzt die Minimal-Drive-Simulation. Firestore nutzt einen deterministischen systemseitigen Fallback fuer den Emulator-Pfad.

Record korrekt

Unit-Tests pruefen die TeamSeasonStat-Inkremente. Der Week-Loop-E2E zeigt danach einen aktualisierten Record im Dashboard.

Verbleibende Hinweise

  • Der Firestore-Emulator muss fuer Firestore-Suites separat gestartet werden.
  • Der UI-Button auf dem Live Screen beendet nicht den Score, sondern fuehrt zum Report und markiert das gespeicherte Simulationsergebnis als abgeschlossen.
  • Keine Game-Engine-Architektur wurde erweitert; die Aenderung bleibt auf Integritaet und Persistenz begrenzt.
