# Reports

## Zweck

Dieser Ordner enthaelt versionierte, kuratierte Reports. Generierte Rohoutputs aus Skripten, Playwright oder Exporten liegen unter `reports-output/`.

## Kategorien

- [phases/](./phases/) - Arbeitspaket- und Phasenberichte.
- [qa/](./qa/) - manuelle oder automatisierte QA-Validierungen.
- [simulations/](./simulations/) - versionierte Simulationsberichte und bewusst abgelegte Ergebnis-Snapshots.
- [systems/](./systems/) - System-, Engine-, Firebase-, Datenmodell- und Analyseberichte.

## Ablageregel

Neue Reports werden nach `docs/guides/documentation-guidelines.md` benannt. Kurzform:

- Phase Report: `phase-<area>-ap<nr>-report.md`
- QA Report: `qa-<system>-check.md`
- Simulation Report: `sim-<scenario>-<runs>-report.md`
- System Report: `<domain>-<topic>-report.md`
