Live Simulation UX

Live Screen Emotionalization

Der Live Screen wurde um eine Storyline-Schicht erweitert, die vorhandene Score-, Drive-, Team- und Lineup-Daten in Spielspannung uebersetzt. Es wurde keine neue Game Engine, keine Play-Calling-Logik und keine neue Simulationsebene gebaut.

Executive Summary

Vorher wirkte der Live Screen stark wie ein Kontrollpanel mit Score-Eingabe, Situation, Timeline und Signalen. Nach der Aenderung fuehrt eine neue Storyline-Schicht den Nutzer durch die emotionale Lesart des Spiels: aktueller Zustand, Schluesseldrive, Momentum, Managerentscheidung und naechster Fokus.

Umgesetzte Storyline-Schicht

Element Nutzen Datenbasis
Current Game State Erklaert, ob das Spiel eng, offen, abgeschlossen oder gekippt ist. Match Status, Score, Teamdaten
Key Drive Hebt den wichtigsten Drive hervor statt nur eine Liste zu zeigen. Drive Summary, Result Type, Yards, Points, Turnover
Momentum Signal Verdichtet die letzten Drives zu Druck, Rhythmus oder Swing. Letzte 3 Drives, Punkte, Yards, Turnovers
Manager Impact Verbindet Lineup-Entscheidungen mit passenden Drives oder Phasen. Recent Decision Events, Depth Chart Parser, Drive Context
What to watch next Gibt dem Nutzer einen klaren Fokus fuer den naechsten Moment. Letzter Drive, Punkte, Turnover, Match Status

UX-Verbesserungen

  • Der Live Screen hat jetzt eine obere Storyline, bevor Details und Controls folgen.
  • Kritische Drives werden als Spielmomente beschrieben: Touchdown, Turnover, Red Zone, Big Gain, Sack.
  • Momentum wird als Druck- oder Rhythmus-Signal formuliert.
  • Managerentscheidungen werden sichtbar, wenn eine Depth-Chart-Aenderung zum Drive passt.
  • Technische Sprache wurde entschaerft: statt sichtbarem Fallback-Fokus gibt es glaubwuerdige Statushinweise wie "Noch keine Drives".

Bekannte Grenzen

  • Keine neue Engine-Logik: Storyline ist eine UI-Interpretation vorhandener Daten.
  • Wenn keine Drives gespeichert sind, bleibt der Screen bewusst bei Score, Status und Beobachtungsfokus.
  • Drive-Daten sind weiterhin Drive-Level, nicht Snap-by-Snap oder Play-Calling.
  • Manager Impact kann nur sichtbar werden, wenn gespeicherte Decision Events und Drive-Kontext zusammenpassen.
  • Keine Chemistry- oder X-Factor-Logik wurde eingefuehrt.
