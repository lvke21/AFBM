# Beobachtungsbogen

Anweisung fuer Beobachter:
- Waerend der Session live ausfuellen.
- Keine Interpretationen ohne beobachtbares Verhalten eintragen.
- Zeiten relativ zum sichtbaren Dashboard messen.
- Wenn ein Feld nicht zutrifft, `n/a` eintragen.
- Beispiele stehen in Klammern und koennen ersetzt werden.

## Sessiondaten

Teilnehmer-Kuerzel: (z. B. U1)

Datum: (z. B. 2026-04-27)

Geraet / Browser: (z. B. MacBook / Chrome)

Football-Erfahrung: (Keine / Ein bisschen / Regelmaessig / Sehr viel)

Manager-Spiel-Erfahrung: (Keine / Ein bisschen / Regelmaessig / Sehr viel)

Sessiondauer: (z. B. 28 Minuten)

Time to First Blocker: (z. B. 04:10; oder `kein Blocker`)

Anzahl Nachfragen: (z. B. 3)

Anzahl Hilfen: (z. B. 0; Moderator soll nicht helfen)

Kompletter Abbruch: Ja / Nein (Ja nur, wenn Core Loop nicht weiter testbar war)

Technische Probleme: (z. B. keine; Login fehlgeschlagen; Button reagiert nicht)

## Scoring

Bewertung je Abschnitt:
- 2 = ohne Hilfe geschafft
- 1 = mit Unsicherheit oder kleiner Hilfe geschafft
- 0 = nicht geschafft oder falsch verstanden

| Abschnitt | Score | Beobachtung |
| --- | ---: | --- |
| Dashboard: naechste Aktion erkannt |  | (z. B. klickt Week Loop ohne Nachfrage) |
| Rolle als GM verstanden |  | (z. B. sagt "Ich manage Aufstellung und Spiel") |
| Team/Roster gefunden |  | (z. B. findet Team ueber Navigation) |
| Depth Chart verstanden |  | (z. B. erkennt Starter/Backup an einer Position) |
| Entscheidungsauswirkung erkannt |  | (z. B. nennt Value, Need oder Impact-Feedback) |
| Spielvorbereitung verstanden |  | (z. B. nennt Readiness oder Gegnerstaerke) |
| Spiel gestartet |  | (z. B. startet ohne Hilfe) |
| Spiel abgeschlossen |  | (z. B. erreicht Report) |
| Post-Game-Report verstanden |  | (z. B. erklaert Ergebnisursache) |
| Naechste Aktion nach Report genannt |  | (z. B. nennt naechste Woche oder Team-Fix) |
| Zweite Woche gestartet oder gewollt |  | (z. B. klickt Naechste Woche oder sagt warum nicht) |

Maximal: 22 Punkte.

## Kritische Momente

- MAJOR: Nutzer trifft eine falsche Entscheidung oder versteht eine zentrale Bedeutung falsch.
- MINOR: Nutzer zoegert, ist unsicher oder braucht Orientierung, kommt aber weiter.
